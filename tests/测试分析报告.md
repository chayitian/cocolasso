### 1.2 测试框架与模式

项目采用**自建测试框架**，未使用 pytest/unittest，而是基于自定义的 `record()` 函数记录每个测试用例的 PASS/FAIL 状态和详细信息，最终输出 JSON 格式的结构化结果。每个测试文件独立运行，结果保存至 `tests/results/` 目录。

### 1.3 测试数据生成

所有测试共享同一数据生成函数 `generate_data()`：

- 生成 n×p 的标准正态矩阵 X，中心化并按列标准化
- 真实系数 β_true 仅前 3 个分量非零（3, 1.5, 2）
- 噪声标准差 σ=1.0
- 加性噪声：Z = X + N(0, τ²)·I
- 缺失数据：Z 中随机 10% 元素设为 NaN
- 乘性噪声：Z = X * exp(N(0, τ²))

### 3.2 2026-05-12 失败项分析

| 失败测试 | 详情 | 根因分析 |
|----------|------|----------|
| `BD_coef_sd - lambda_sd_ >= lambda_opt_` | lambda_sd=0.017825 < lambda_opt=0.053053 | 1-std 准则在 BD 模块中选择逻辑有误：候选 lambda 应满足 ≥ 最优 lambda，但实际选到了更小的值 |
| `General_3block - score(R²)有限` | R²=nan | 三块混合场景下 predict 产生 NaN，可能因缺失数据块处理后残存 NaN |
| `General_coef_sd - lambda_sd_ >= lambda_opt_` | lambda_sd=0.017825 < lambda_opt=0.053053 | 同 BD 模块的 1-std 准则问题 |
| `timing - 结果一致性` | max_diff=0.180329 > 0.2 容差 | CD 与 sklearn 在较大数据集上的 CV 过程放大了数值差异 |

### 3.3 修复评估

2026-05-13 的测试全部通过，说明上述 4 个问题已修复。主要修复方向推测为：

1. **1-std 准则逻辑修正**：`lambda_sd_` 的选择条件从"最接近最优误差+1std 的 lambda"改为"满足误差 ≤ 阈值且 lambda ≥ 最优 lambda 的最小 lambda"
2. **NaN 处理增强**：GeneralCoCoLasso 的 predict 方法中增加了 NaN→0 的替换
3. **求解器容差放宽**：timing 测试中 max_diff=0.180329 仍在 0.2 容差内通过

## 4. 测试质量评估

### 4.1 优势

1. **覆盖层次完整**：从底层工具函数（单元测试）到完整估计器（集成测试）再到求解器一致性，层次分明
2. **三种噪声类型全覆盖**：加性、缺失、乘性噪声均有独立测试
3. **sklearn API 兼容性验证**：继承、get_params/set_params 均有测试
4. **数值正确性验证**：乘性校正公式与手算精确对比，PSD 投影的特征值验证
5. **结果可追溯**：JSON 格式保存每次运行的完整结果，便于历史对比

### 4.2 不足与建议

| 问题 | 严重程度 | 建议 |
|------|----------|------|
| 无 pytest/unittest 框架 | 中 | 迁移至 pytest 可获得更好的断言报告、fixture 管理、参数化测试 |
| 测试数据规模偏小（n≤80, p≤40） | 中 | 增加中大规模测试（n=500, p=200）验证可扩展性 |
| 缺少边界条件测试 | 中 | 增加 tau=0、p=1、n=p 等边界场景 |
| 缺少回归测试 | 低 | 对已知 bug（如 1-std 准则）添加回归测试 |
| 缺少随机性控制 | 低 | CV 折分配使用 np.random.permutation 但未固定全局种子，不同运行可能产生不同折划分 |
| 缺少性能基准 | 低 | 添加 CI 可运行的性能基准测试 |
| BDCoCoLasso 不支持乘性噪声 | 信息 | 测试中未覆盖 BD+乘性噪声组合，代码也不支持 |
| GeneralCoCoLasso 不支持乘性噪声 | 信息 | 三块模型仅支持加性+缺失组合 |
